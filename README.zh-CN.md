## Paperclip 是什么？

# 零人力公司的开源编排系统

**如果说 OpenClaw 是_员工_，那么 Paperclip 就是_公司_**

Paperclip 是一个 Node.js 服务器和 React UI，用于编排 AI 代理团队来运营企业。自带代理、分配目标，并通过一个仪表盘跟踪代理的工作和成本。

它看起来像是一个任务管理器 —— 但在底层，它有组织架构、预算、治理、目标对齐和代理协调。

**管理业务目标，而不是 Pull Request。**

|        | 步骤            | 示例                                                            |
| ------ | --------------- | ------------------------------------------------------------------ |
| **01** | 定义目标 | _"打造排名第一的 AI 笔记应用，实现 $100万 MRR。"_                    |
| **02** | 组建团队   | CEO、CTO、工程师、设计师、市场人员 —— 任何机器人，任何提供商。 |
| **03** | 审批并运行 | 审阅策略。设定预算。点击开始。通过仪表盘监控。  |

<br/>

> **即将推出：Clipmart** — 一键下载并运行完整的公司。浏览预置的公司模板 —— 完整的组织架构、代理配置和技能 —— 并在几秒钟内导入到你的 Paperclip 实例中。

## Paperclip 适合你，如果你

- ✅ 想要构建**自主 AI 公司**
- ✅ 需要**协调多个不同的代理**（OpenClaw、Codex、Claude、Cursor）实现共同目标
- ✅ 有**20 个同时运行的 Claude Code 终端**开着，但无法跟踪每个人都在做什么
- ✅ 希望代理**全天候 24/7 自主运行**，但仍希望审核工作并在需要时参与
- ✅ 想要**监控成本**并强制执行预算
- ✅ 想要一个**感觉像使用任务管理器**的代理管理流程
- ✅ 想要通过手机**管理你的自主企业**

## Paperclip 解决的问题

| 没有 Paperclip                                                                                                                     | 使用 Paperclip                                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| ❌ 你有 20 个 Claude Code 标签页开着，无法跟踪哪个做什么。重启后一切丢失。                              | ✅ 任务基于工单，对话有线程，会话在重启后持久化。                                                |
| ❌ 你手动从多个地方收集上下文，提醒机器人你实际在做什么。                                     | ✅ 上下文从任务流经项目和公司目标 —— 你的代理始终知道做什么以及为什么。                  |
| ❌ 代理配置文件夹杂乱无章，你在重复发明任务管理、通信和代理之间的协调。 | ✅ Paperclip 开箱即提供组织架构、工单、委托和治理 —— 这样你运营的是公司，而不是一堆脚本。 |
| ❌ 失控的循环浪费数百美元的 Token，在你察觉之前就耗尽了配额。                           | ✅ 成本追踪显示 Token 预算，并在超支时限制代理。管理层按预算优先处理。                    |
| ❌ 你有定期工作（客户支持、社交、报告），必须记住手动启动。                        | ✅ 心跳机制按计划处理常规工作。管理层监督。                                                                |
| ❌ 你有一个想法，必须找到你的仓库，启动 Claude Code，保持一个标签页打开，并看护它。                                | ✅ 在 Paperclip 中添加任务。你的编码代理完成它。管理层审阅他们的工作。                              |

## Paperclip 的独特之处

Paperclip 正确处理困难的编排细节。

|                                   |                                                                                                               |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **原子执行。**             | 任务签出和预算执行是原子的，因此没有重复工作，也没有失控支出。                      |
| **持久化代理状态。**       | 代理在心跳之间恢复相同的任务上下文，而不是从头重新开始。                     |
| **运行时技能注入。**      | 代理可以在运行时学习 Paperclip 工作流程和项目上下文，无需重新训练。                      |
| **带回滚的治理。**     | 强制执行审批门控，配置更改有版本控制，错误更改可以安全回滚。        |
| **目标感知执行。**         | 任务携带完整的目标祖先，因此代理始终看到"为什么"，而不仅仅是标题。                        |
| **可移植的公司模板。**   | 导出/导入组织、代理和技能，包含秘密清理和冲突处理。                          |
| **真正的多公司隔离。** | 每个实体都是公司范围的，因此一次部署可以运行多家公司，拥有独立的数据和审计跟踪。 |

## Paperclip 不是什么

|                              |                                                                                                                      |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **不是聊天机器人。**           | 代理有工作，没有聊天窗口。                                                                                  |
| **不是代理框架。**  | 我们不告诉你如何构建代理。我们告诉你如何运营一家由它们组成的公司。                                |
| **不是工作流构建器。**  | 没有拖放管道。Paperclip 建模公司 —— 带有组织架构、目标、预算和治理。            |
| **不是提示管理器。**    | 代理自带提示词、模型和运行时。Paperclip 管理它们工作的组织。               |
| **不是单代理工具。** | 这是为团队准备的。如果你只有一个代理，你可能不需要 Paperclip。如果你有二十个 —— 你肯定需要。                                       |
| **不是代码审查工具。**  | Paperclip 编排工作，而不是 Pull Request。自带审查流程。                                       |

## 快速开始

开源。自托管。不需要 Paperclip 账户。

```bash
npx paperclipai onboard --yes
```

该快速开始路径现在默认为受信任的本地环回模式，以获得最快的首次运行体验。要以认证/私有模式启动，请显式选择绑定预设：

```bash
npx paperclipai onboard --yes --bind lan
# 或：
npx paperclipai onboard --yes --bind tailnet
```

如果你已经配置了 Paperclip，重新运行 `onboard` 会保留现有配置。使用 `paperclipai configure` 编辑设置。

或者手动：

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

这会在 `http://localhost:3100` 启动 API 服务器。嵌入式 PostgreSQL 数据库会自动创建 —— 无需设置。

> **要求：** Node.js 20+，pnpm 9.15+

## 常见问题

**典型设置是什么样的？**
本地，单个 Node.js 进程管理嵌入式 Postgres 和本地文件存储。对于生产环境，指向你自己的 Postgres 并按你喜欢的方式部署。配置项目、代理和目标 —— 代理处理其余部分。

如果你是个人创业者，可以使用 Tailscale 在移动中访问 Paperclip。然后当你需要时，可以部署到例如 Vercel。

**可以运行多家公司吗？**
可以。一次部署可以运行无限数量的公司，完全数据隔离。

**Paperclip 与 OpenClaw 或 Claude Code 等代理有什么不同？**
Paperclip _使用_ 那些代理。它将它们编排成一家公司 —— 带有组织架构、预算、目标、治理和问责制。

**为什么我应该使用 Paperclip 而不是直接将我的 OpenClaw 指向 Asana 或 Trello？**
代理编排在工作签出协调、会话维护、成本监控、治理建立等方面有微妙之处 —— Paperclip 为你处理这些。

（自带工单系统已在路线图中）

**代理会持续运行吗？**
默认情况下，代理按计划心跳和基于事件的触发器（任务分配、@提及）运行。你还可以接入连续代理如 OpenClaw。你带来代理，Paperclip 进行协调。

## 开发

```bash
pnpm dev              # 完整开发（API + UI，监听模式）
pnpm dev:once         # 完整开发，无文件监听
pnpm dev:server       # 仅服务器
pnpm build            # 构建全部
pnpm typecheck        # 类型检查
pnpm test             # 默认测试运行（仅 Vitest）
pnpm test:watch       # Vitest 监听模式
pnpm test:e2e         # Playwright 浏览器套件
pnpm db:generate      # 生成数据库迁移
pnpm db:migrate       # 应用迁移
```

`pnpm test` 不运行 Playwright。浏览器套件保持独立，通常只在处理这些流程或 CI 时运行。

完整开发指南参见 [doc/DEVELOPING.md](doc/DEVELOPING.md)。

## 遥测

Paperclip 收集匿名使用遥测数据，以帮助我们了解产品使用情况并改进它。从不收集个人信息、问题内容、提示词、文件路径或机密。私有仓库引用在发送前会使用每安装盐值进行哈希处理。

遥测**默认启用**，可以通过以下任一方式禁用：

| 方法               | 如何操作                                                     |
| -------------------- | ------------------------------------------------------- |
| 环境变量 | `PAPERCLIP_TELEMETRY_DISABLED=1`                        |
| 标准约定  | `DO_NOT_TRACK=1`                                        |
| CI 环境      | `CI=true` 时自动禁用                   |
| 配置文件          | 在你的 Paperclip 配置中设置 `telemetry.enabled: false` |
